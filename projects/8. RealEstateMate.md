### 1. Project Title
**RealEstateMate: AI-Powered Real Estate Concierge**

### 2. Real-World Context / Business Background
Finding a home is high-stakes and emotional. Traditional real estate platforms (like Zillow or Housing.com) rely heavily on rigid filters (e.g., "3 BHK," "Price < $2000"). However, users often have abstract requirements like *"I want a home with lots of natural light for my plants"* or *"A quiet neighborhood suitable for remote work."* Standard databases cannot filter for "Vibe" or "Natural Light." Additionally, real estate agents waste hours answering repetitive questions about lease terms or amenities that are buried in long property descriptions.

### 3. Problem Statement
Develop **RealEstateMate**, a smart property listing platform. The application allows agents to list properties and users to search for them. The core innovation is **Semantic Search**, allowing users to find homes based on lifestyle descriptions rather than just checkboxes. Furthermore, the system must use **RAG** to power a "Property Chatbot" that answers specific questions about a listing (e.g., *"Is this pet-friendly?"*) by retrieving details from the specific property's description and lease documents.

### 4. Target Users
- **Home Buyers/Renters:** Who want a more intuitive search experience.
- **Real Estate Agents:** Who need help writing listing descriptions and answering user queries automatically.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Lifestyle-Based Property Search (Semantic Search)**
    *   Users can search using natural language phrases like *"A bachelor pad with a great view of the city"* or *"Close to parks for my dog."*
    *   The system queries **Pinecone** to find property descriptions that semantically match these needs, returning results that strict keyword filters would miss.
2.  **Automated Listing Description Generator**
    *   When an Agent uploads a property, they input raw features (e.g., "3 Bed, 2 Bath, Pool, $250k"). The AI generates a compelling, marketing-ready paragraph describing the property to attract buyers (e.g., *"Step into luxury with this stunning 3-bedroom..."*).
3.  **"Ask the House" (RAG Q&A)**
    *   On every property detail page, there is a chat widget. A user can ask, *"Is the kitchen gas or electric?"* or *"What schools are nearby?"*
    *   The system uses RAG to fetch the specific property's description and metadata to answer accurately without the agent's intervention.
4.  **Lease Term Summarizer**
    *   Agents can upload a standard rental agreement text. The AI analyzes it and displays a "Quick Look" card for tenants, highlighting key terms: *Security Deposit amount, Notice Period, Pet Policy, and Hidden Fees*.
5.  **Investment Potential Score**
    *   Based on the property price and a hardcoded "Average Rental Yield" for the area in the database, the AI generates a brief analysis: *"Good Investment: This property is priced 10% below market average and has high rental potential."*

### 6. High-Level System Architecture

*   **Frontend (React):** A property listing grid, a detailed "Single Property" view with an image carousel, and a floating Chat widget.
*   **Backend (Node.js + Express):** APIs for listing management (CRUD) and handling AI search/chat requests.
*   **Database (MongoDB Atlas):**
    *   Stores: Structured Property Data (Address, Price, Square Footage, Number of Rooms), Agent Profiles, and raw Listing Text.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of the **Property Description** and **Amenities List**.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Generating marketing copy, Vectorizing search queries, and RAG-based chat responses.

### 7. AI & RAG Workflow Explanation

**Phase A: Listing Creation (Ingestion)**
1.  **Agent Input:** Agent fills a form: "Downtown Apt, $1500, Gym, Balcony".
2.  **Generation:** Agent clicks "Generate Description." Gemini writes a paragraph. Agent saves.
3.  **Embedding:** Node.js sends the generated description to Gemini to get a vector.
4.  **Storage:**
    *   *MongoDB:* Saves the full listing object.
    *   *Pinecone:* Saves the vector with metadata `{ "propertyId": "123", "price": 1500 }`.

**Phase B: "Ask the House" (The RAG Flow)**
1.  **User Question:** User is viewing Property A and asks: *"Is there parking available?"*
2.  **Retrieval:** The system fetches the full description and amenity list for **Property A** from MongoDB (Note: For a single property chat, we might not need Pinecone if we already have the specific document ID, but we can use Pinecone if searching across documents. Here, direct MongoDB fetch is acceptable for specific context).
3.  **Augmentation:**
    *   *Context:* (Text of Property A).
    *   *Question:* "Is there parking?"
    *   *Instruction:* "Answer strictly based on the context. If not mentioned, say 'Please contact the agent'."
4.  **Generation:** Gemini generates: *"Yes, the listing mentions an underground reserved parking spot."*

### 8. Expected Learning Outcomes
*   Differentiating between **Structured Search** (MongoDB filters for Price) vs **Semantic Search** (Pinecone for "Vibe").
*   Implementing **Context-Specific Chatbots** (The chatbot knows which house the user is looking at).
*   Using AI to **Augment Content Creation** (Writing descriptions for users).
*   Handling **Hybrid Search** UI (Combining a text search bar with dropdown filters).

### 9. Constraints & Assumptions
*   **Images:** Students can use placeholder images (e.g., Unsplash URLs) for properties. No complex image hosting needed.
*   **Map:** No Google Maps API integration required; just displaying the text address is sufficient.
*   **Data:** Seed the database with 10-15 mock property listings.

### 10. Optional Extensions (For Advanced Students)
*   **Similar Property Recommender:** "If you like this house, you might also like..." (Use Pinecone to find the nearest vector neighbors to the current property).
*   **Multi-Language Listings:** Allow the user to view the property description in Spanish or Hindi (Real-time AI translation).
*   **Negotiation Email Drafter:** A button "Make an Offer" that asks the user for a price and uses AI to write a professional email to the agent arguing why that price is fair.
