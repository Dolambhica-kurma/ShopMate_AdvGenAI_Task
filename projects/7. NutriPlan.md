### 1. Project Title
**NutriPlan AI: Smart Meal Planner & Pantry Assistant**

### 2. Real-World Context / Business Background
People often struggle to maintain a healthy diet not because of a lack of will, but because of decision fatigue. Questions like *"What can I cook with these three ingredients in my fridge?"* or *"How do I eat high-protein while being Vegetarian?"* are common. Generic recipe apps filter by tags, but they don't understand context or specific health goals. Users need a personalized assistant that understands their dietary needs and their available inventory.

### 3. Problem Statement
Build **NutriPlan AI**, a nutrition assistant application. The system allows users to manage a profile of dietary preferences (e.g., Vegan, Keto, Allergies) and available pantry ingredients. The core functionality uses **RAG** to generate meal plans strictly from a curated database of verified healthy recipes.

> **Why RAG?** Unlike a general LLM which might hallucinate recipes with incorrect ratios or undetectable ingredients, a curated database ensures users get **proven, high-quality recipes** while still benefiting from AI's ability to search by "vibe" or context.

Additionally, it should leverage **Gemini’s Multimodal capabilities** to recognize ingredients from photos.

### 4. Target Users
- **Fitness Enthusiasts:** Tracking macros and protein intake.
- **Home Cooks:** Who want to reduce food waste by using what they have.
- **Busy Professionals:** needing quick, healthy meal decisions.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **"Snap & Cook" (Multimodal Ingredient Recognition)**
    *   Allow users to upload a photo of items on their kitchen counter. Use **Gemini Vision (Multimodal)** to identify the ingredients (e.g., "I see Eggs, Spinach, and Tomatoes"), auto-add them to the user's "Virtual Pantry," and suggest recipes.
2.  **Semantic "Vibe" Search (Recipe Discovery)**
    *   Search for recipes based on abstract needs or moods. Example: *"I need a comforting dinner for a cold rainy day."* The system queries Pinecone to find recipes with descriptions matching "comforting," "warming," or "soup," even if the words don't match exactly.
3.  **Goal-Oriented Meal Planner (RAG)**
    *   Generate a **1-Day Meal Plan** (Breakfast, Lunch, Dinner). The AI must select recipes *only* from the curated database that match the user's calorie/diet goals and filter for ingredients they (mostly) have.
4.  **Smart Shopping List Generator**
    *   Once a meal plan is generated, compare the required ingredients against the user's Pantry. Use AI to generate a structured shopping list of *only* the missing items, grouped by category.
5.  **Health-ify My Plate**
    *   A one-click feature where the AI analyzes the selected recipe and rewrites the ingredients/steps to be healthier (e.g., swapping "Sugar" for "Honey," "Frying" for "Baking") while maintaining the flavor profile.


### 6. High-Level System Architecture

*   **Frontend (React):** A responsive UI with an image upload zone, a recipe card grid, a "My Pantry" list, and a chat interface for nutrition advice.
*   **Backend (Node.js + Express):** Handles image processing (sending to Gemini), database queries, and managing user sessions.
*   **Database (MongoDB Atlas):**
    *   Stores: User Profile (Diet tags), Virtual Pantry (List of strings), and Recipe Data (Ingredients, Instructions, Macros).
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Recipe Descriptions** and **Flavor Profiles**.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Image recognition (Gemini Pro Vision), Embedding generation, and Planning/Chat logic.

### 7. AI & RAG Workflow Explanation

**Phase A: Recipe Database Creation**
1.  **Ingestion:** Seed the database with 20 healthy recipes (JSON format).
2.  **Embedding:** Create a text chunk for each recipe containing its title, description, and key tags (e.g., "Spicy Chicken Curry, high protein, warming, dinner").
3.  **Storage:** Store vectors in Pinecone.

**Phase B: Meal Planning (The RAG Flow)**
1.  **User Request:** "Plan a high-protein lunch using the chicken in my pantry."
2.  **Inventory Check:** Backend retrieves "Chicken" from the user's MongoDB Pantry.
3.  **Retrieval:** Query Pinecone for recipes that:
    *   Semantically match "high protein" + "lunch".
    *   (Optional Metadata Filter) contain "Chicken".
4.  **Augmentation:**
    *   *Context:* (Full details of top 3 matching recipes).
    *   *User Goal:* "High protein, use chicken."
    *   *Task:* "Select the best option and explain why it fits the goal."
5.  **Generation:** Gemini returns the selected recipe and a brief explanation ("I chose the Grilled Chicken Salad because it has 30g protein...").

### 8. Expected Learning Outcomes
*   Working with **Multimodal AI models** (handling images vs text).
*   Implementing **filtering logic in Vector Search** (e.g., ensuring a Vegan user never sees a steak recipe).
*   Managing **User Context (State)** across the application (Pantry + Profile + AI Goals).
*   Structuring **Complex Prompts** that involve inventory comparison (Have vs Need).

### 9. Constraints & Assumptions
*   **Recipe Data:** Students can use a small, static JSON file of ~20 recipes to seed the database (no need for an external Recipe API).
*   **Medical Disclaimer:** The app must include a UI disclaimer that "AI suggestions are not medical advice."
*   **Image Upload:** Use a simple file input in React; no need for live camera integration.

### 10. Optional Extensions (For Advanced Students)
*   **Leftover Saver:** A specific feature where the user inputs "I have 1/2 onion and a lemon, what can I make?" and the AI finds a recipe to minimize waste.
*   **Step-by-Step Cooking Mode:** A chat mode where the AI reads one instruction at a time and waits for the user to say "Next", answering questions like "How do I mince garlic?" in between.
*   **Calorie Vision:** Estimate the calories of a meal simply by uploading a photo of the finished plate (using Gemini Vision).
