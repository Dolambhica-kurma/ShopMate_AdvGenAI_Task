### 1. Project Title
**DocuMinds: AI-Powered Contract & Document Assistant**

### 2. Real-World Context / Business Background
Freelancers, small business owners, and startup founders often deal with complex contracts (NDAs, Service Agreements, Leases) but cannot afford expensive legal counsel for every document. They often sign agreements without fully understanding the "fine print" or missing critical deadlines buried in the text. There is a need for a tool that can quickly "translate" legalese into plain English and highlight potential risks automatically.

### 3. Problem Statement
Build **DocuMinds**, a document intelligence dashboard. The application allows users to upload legal documents or agreements. Using **RAG (Retrieval-Augmented Generation)**, the system must index the document content to allow users to ask specific questions (e.g., *"What is the notice period for termination?"*). Crucially, the system must proactively analyze the document to flag "High Risk" clauses and extract actionable data like dates and financial obligations into a structured table.

### 4. Target Users
- **Freelancers/Consultants:** Reviewing client contracts.
- **Startup Founders:** managing multiple vendor agreements.
- **Admin Teams:** Who need to track renewal dates and obligations.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **"Explain Like I'm 5" (ELI5) Summarizer**
    *   Generate a summary of the uploaded document, but specifically "translated" from legal jargon to simple, everyday language.
2.  **Automated Risk & Red Flag Detection**
    *   Analyze the document against a set of standard safety guidelines (in the prompt). The AI should identify and list "Risky Clauses" (e.g., Unlimited Liability, Non-Compete for 5 years) with a severity score (High/Medium/Low).
3.  **Semantic Document Search (Cross-Document)**
    *   Allow the user to search across *all* their uploaded documents. Example: *"Which of my contracts have a confidentiality clause?"* The system uses Pinecone to find matches across the entire database.
4.  **Key Entitiy Extraction (Dates & Money)**
    *   Use Gemini to extract structured data: Effective Dates, Expiry Dates, Payment Amounts, and Penalties. This must be returned as a JSON array to populate a "Key Dates" calendar or table in the UI.
5.  **Context-Aware Clause rewriting**
    *   If a user highlights a specific paragraph, use AI to suggest a "Friendlier Version" or a "More Protective Version" of that specific clause to help in negotiations.

### 6. High-Level System Architecture

*   **Frontend (React):** A document viewer (split screen: text on left, AI insights on right), a dashboard for all files, and a search bar.
*   **Backend (Node.js + Express):** Handles file uploads, text chunking, and communication with AI services.
*   **Database (MongoDB Atlas):**
    *   Stores: User metadata, Document Metadata (Title, Date Uploaded, Category), and the *Structured Analysis Results* (JSON output from Feature #4).
*   **Vector Database (Pinecone):**
    *   Stores: Text chunks of the documents transformed into embeddings. This enables the RAG functionality for Q&A and Search.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Summarization, Risk Analysis, Entity Extraction, and Rewriting.

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion & Indexing**
1.  **Upload:** User uploads a contract (Text/PDF).
2.  **Chunking:** Node.js breaks the text into smaller chunks (e.g., 500 characters or by paragraph).
3.  **Embedding:** Each chunk is sent to Gemini to get a vector embedding.
4.  **Storage:**
    *   *Pinecone:* Stores vectors + metadata (Document ID, Chunk ID).
    *   *MongoDB:* Stores the full text and file metadata.

**Phase B: Q&A / Analysis (The RAG Flow)**
1.  **User Query:** User asks: *"Can I work for other clients while on this contract?"*
2.  **Retrieval:** The system embeds the question and queries Pinecone for the 3 most relevant text chunks from the document.
3.  **Augmentation:** Construct prompt:
    *   *Context:* (The text of the retrieved chunks regarding "Exclusivity" or "Non-Compete").
    *   *Question:* "Can I work for other clients?"
4.  **Generation:** Gemini answers the question citing the specific clause found in the context.

### 8. Expected Learning Outcomes
*   Implementing **Text Chunking strategies** (splitting long documents so they fit into AI context windows).
*   Using **Vector Databases** for specific document retrieval (Document Q&A).
*   Practicing **Structured Data Extraction** to turn unstructured text into database rows (Dates/Money).
*   Building a **Split-view UI** in React to handle document visualization alongside AI chat.

### 9. Constraints & Assumptions
*   **Input Method:** To keep it simple, allow users to "Paste Text" or upload simple `.txt` files. If using PDF, use a basic library like `pdf-parse` on the backend.
*   **Document Length:** Assume documents are short to medium length (2-10 pages).
*   **Legal Disclaimer:** The UI must display a disclaimer that "AI is not a lawyer" (Good practice for real-world product design).

### 10. Optional Extensions (For Advanced Students)
*   **Comparison Mode:** Select two documents (e.g., "Version 1" and "Version 2") and ask the AI to list exactly what changed and how it affects the user.
*   **Email Generation:** A button to "Email Lawyer" that drafts an email summarizing the user's concerns and attaching the document.
*   **Category Classification:** Auto-tag documents as "NDA", "Invoice", "Employment" upon upload using Gemini classification.
