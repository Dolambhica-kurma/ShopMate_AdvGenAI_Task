### 1. Project Title
**CourseWhiz: AI-Powered Study Companion & Quiz Engine**

### 2. Real-World Context / Business Background
Students and corporate upskillers often have access to massive amounts of learning material (PDF textbooks, lecture notes, documentation). However, passively reading is the least effective way to learn. Creating manual flashcards, practice quizzes, and finding specific answers within 500-page documents is time-consuming. Learners need a tool that transforms static content into an active, interactive learning experience to verify their understanding before exams.

### 3. Problem Statement
Build **CourseWhiz**, a learning management tool where users can create "Courses" by uploading their own reference materials (notes/chapters). The system must use **RAG** to ensure all answers and quizzes are strictly based on the uploaded material, not general internet knowledge. A key differentiator is the **AI Grading System**, which evaluates subjective (written) answers against the source text and provides constructive feedback.

### 4. Target Users
- **University Students:** Preparing for semester exams using professor's notes.
- **Corporate Trainees:** Learning internal company policies or technical manuals.
- **Self-Learners:** Studying from whitepapers or documentation.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Interactive "Chat with Material"**
    *   Users can ask specific questions like *"What does the author say about Quantum Entanglement?"*. The system uses Vector Search to retrieve the specific paragraph from the uploaded notes and answers based *only* on that context.
2.  **Automated Quiz Generator (MCQ & Short Answer)**
    *   One-click generation of a 5-question quiz based on the uploaded content. The AI must output structured JSON (Question, Options, Correct Answer) to render a proper quiz UI.
3.  **Subjective Answer Grading & Feedback**
    *   For short-answer questions, the user types a response. The AI compares the User's Answer vs. the Source Text (Ground Truth) and assigns a score (e.g., 7/10) with feedback on what key points were missed.
4.  **Smart Flashcard Creation**
    *   Scan the document for complex terminology or definitions and automatically generate "Front/Back" flashcard content in a JSON format for revision.
5.  **Topic Summarizer & Key Takeaways**
    *   Generate a "Cheat Sheet" summary of the document, breaking it down into: *Core Concepts, Key Dates/Formulas, and Actionable Steps*.

### 6. High-Level System Architecture

*   **Frontend (React):** A dashboard showing "My Courses." Inside a course: tabs for "Study" (Chat), "Quiz" (Assessment), and "Flashcards."
*   **Backend (Node.js + Express):** APIs for file processing, quiz generation, and grading logic.
*   **Database (MongoDB Atlas):**
    *   Stores: User progress (Quiz scores), Generated Flashcards, and Course Metadata.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of the study materials (chunked text). This is the "Brain" of the course.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Generating Embeddings, Creating Quizzes (JSON), and Grading answers.

### 7. AI & RAG Workflow Explanation

**Phase A: Knowledge Ingestion**
1.  **Course Creation:** User creates a course "History 101" and pastes Chapter 1 text.
2.  **Processing:** Node.js cleans the text and splits it into logical chunks.
3.  **Embedding:** Gemini converts chunks into vectors.
4.  **Storage:** Vectors stored in Pinecone with `courseId` as metadata (to ensure searches stay within the correct course).

**Phase B: The "AI Grader" Workflow (RAG)**
1.  **Quiz Generation:** User clicks "Take Quiz." AI generates a question: *"Explain the cause of the event described in paragraph 3."*
2.  **User Attempt:** User types: *"It was caused by economic instability."*
3.  **Retrieval:** System fetches the actual text paragraph regarding the event from Pinecone.
4.  **Evaluation (The Prompt):**
    *   *Context:* (Actual Paragraph Text)
    *   *Question:* "Explain the cause..."
    *   *Student Answer:* "It was caused by economic instability."
    *   *Task:* "Grade the answer 0-10 based strictly on the context. If incorrect, explain why."
5.  **Output:** React UI displays the Score: **4/10** and Feedback: *"Partially correct, but you missed the political factors mentioned in the text."*

### 8. Expected Learning Outcomes
*   Implementing **Scoped Search** (filtering Pinecone queries by `courseId` so the AI doesn't mix up History and Biology notes).
*   Handling **JSON Output parsing** from LLMs to render interactive UI elements (Quizzes).
*   Building an **Evaluation Pipeline** (AI checking human work).
*   State management in React for Quiz navigation and Score tracking.

### 9. Constraints & Assumptions
*   **Input Scope:** Limit to text-based inputs (Copy-paste) for the workshop to avoid parsing complex PDF images/diagrams.
*   **Quiz Size:** Limit generation to 5 questions at a time to manage API token limits.
*   **Content:** Students should use Wikipedia articles or public domain text as "Study Material."

### 10. Optional Extensions (For Advanced Students)
*   **Difficulty Adjustment:** Allow users to select "Easy," "Medium," or "Hard." The prompt changes to ask for deeper analysis vs simple recall.
*   **Spaced Repetition:** Store which flashcards the user got wrong and show them more frequently (logic in Node.js).
*   **Gamification:** Add a leaderboard or "XP points" based on quiz scores stored in MongoDB.
