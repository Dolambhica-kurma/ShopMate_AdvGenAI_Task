### 1. Project Title
**PennyWise AI: Smart Expense Tracker & Financial Coach**

### 2. Real-World Context / Business Background
Young professionals and students often struggle to manage their finances. While there are many apps to log expenses, they usually require tedious manual categorization (e.g., selecting "Food" for every coffee transaction). Furthermore, looking at a pie chart doesn't tell a user *how* to save money. Users need a system that not only automates the boring data entry but also acts as a "Financial Analyst," offering personalized advice based on their actual spending habits.

### 3. Problem Statement
Build **PennyWise AI**, a personal finance dashboard. The application should allow users to input raw transaction descriptions (e.g., *"Starbucks $5"* or *"Uber Trip $12"*). The system must use **LLM-based Classification** to automatically categorize these expenses without manual rules. The core innovation is a **RAG-based Financial Assistant** that allows users to query their own financial history using natural language (e.g., *"How much did I spend on junk food last month?"*) and receive advice on where to cut costs.

### 4. Target Users
- **Students/Freelancers:** managing tight budgets.
- **Young Professionals:** wanting to track lifestyle inflation.
- **Small Business Owners:** needing a quick overview of expenses.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **AI Auto-Categorization (Intelligent Tagging)**
    *   When a user adds a transaction (e.g., "Payment to SPOTIFY AB"), Gemini analyzes the text and automatically assigns a Category (Entertainment), a Sub-Category (Music Streaming), and a "Necessity Tag" (Want vs. Need).
2.  **"Talk to Your Money" (Semantic Transaction Search)**
    *   Users can ask questions like *"How much have I spent on coffee shops?"*
    *   The system uses **Pinecone** to find transactions that semantically match "coffee" (identifying "Starbucks", "Costa", "Tim Hortons") even if the word "coffee" isn't in the transaction name.
3.  **Spending Personality Analysis**
    *   At the end of the month, the AI analyzes the user's spending patterns and generates a "Financial Personality Profile" (e.g., *"The Impulsive Shopper"* or *"The Savvy Saver"*) along with a text summary explaining *why* (e.g., *"You spent 40% of your income on dining out this week"*).
4.  **Subscription/Recurring Payment Detector**
    *   Analyze the transaction history to identify potential subscriptions. The AI looks for patterns and flagged merchant names, alerting the user: *"I noticed you are paying $15 to Netflix every month. Is this active?"*
5.  **Smart Budget Recommendations (RAG)**
    *   Based on the user's past 3 months of data, the AI suggests realistic budget caps. It explains the logic: *"I suggest a dining budget of $200 because you averaged $250 recently, and we want to cut back slightly."*

### 6. High-Level System Architecture

*   **Frontend (React):** A clean dashboard with a transaction list, a "Add Expense" input (text or voice), charts, and a Chatbot interface.
*   **Backend (Node.js + Express):** APIs for handling transactions, aggregation logic, and AI service calls.
*   **Database (MongoDB Atlas):**
    *   Stores: User Transaction Data (Date, Amount, Merchant, AI-assigned Category), User Budget Settings.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Transaction Descriptions**. This enables the semantic search for "junk food" or "transport".
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Zero-shot classification (Categorization), Insight generation, and Chat interactions.

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion & Classification**
1.  **User Input:** User enters: *"Paid 1500 to cult.fit gym"*.
2.  **Analysis:** Node.js sends text to Gemini with a prompt: *"Categorize this transaction into [Food, Travel, Health, Bills]."
3.  **Result:** Gemini returns JSON: `{ category: "Health", type: "Gym", recurring_likely: true }`.
4.  **Embedding:** The description *"Paid 1500 to cult.fit gym"* is embedded via Gemini.
5.  **Storage:**
    *   *MongoDB:* Stores the full transaction + JSON tags.
    *   *Pinecone:* Stores the vector.

**Phase B: "Talk to Your Money" (The RAG Flow)**
1.  **User Query:** *"How much did I waste on partying?"*
2.  **Retrieval:** The system embeds "partying" and queries Pinecone.
3.  **Match:** It retrieves transactions like "Nightclub Entry," "Bar & Grill," "Liquor Store" (semantic matches).
4.  **Aggregation:** The backend sums the amounts of these retrieved transactions (e.g., Total: $450).
5.  **Generation:** Gemini generates a response: *"Based on your history, you spent $450 on nightlife activities across 3 transactions. This is 15% of your monthly budget."*

### 8. Expected Learning Outcomes
*   Using LLMs for **Data Labeling/Classification** (a very common real-world use case).
*   Performing **Aggregations** (Sum/Average) on top of Vector Search results.
*   Visualizing **AI-categorized data** using Charts (e.g., Recharts or Chart.js).
*   Prompt Engineering for **Financial Advice** (ensuring the tone is helpful, not judgmental).

### 9. Constraints & Assumptions
*   **Currency:** Default to a single currency (e.g., USD or INR) to avoid conversion complexities.
*   **Data Entry:** Users will manually type transactions or paste a CSV string. No Open Banking API integration is needed.
*   **Privacy:** Since this is a student project, use dummy financial data.

### 10. Optional Extensions (For Advanced Students)
*   **Voice Expense Logging:** Use the browser's Speech-to-Text API so users can just say *"Spent 10 bucks on a burger"* and the AI logs it automatically.
*   **"Savings Challenge" Mode:** The AI challenges the user: *"If you skip one coffee a week, you can save $20. Want to try?"*
*   **Receipt Scanner:** Upload an image of a receipt, use Gemini Vision to extract the Total Amount and Merchant Name.
