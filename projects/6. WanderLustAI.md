### 1. Project Title
**WanderLust AI: Hyper-Personalized Smart Travel Planner**

### 2. Real-World Context / Business Background
Planning a vacation is often stressful and time-consuming. Travelers have to scour dozens of blogs, review sites, and booking platforms to piece together an itinerary. Generic travel agencies offer "cookie-cutter" packages that don't account for individual preferences (e.g., a traveler who loves local food but hates museums). Modern travelers want a "Travel Concierge" experience that understands their specific "vibe" and creates a custom plan instantly.

### 3. Problem Statement
Develop **WanderLust AI**, a travel planning application. The system should manage a database of travel destinations (cities, landmarks, hotels, restaurants). Using **Semantic Search**, users should be able to find destinations based on vague descriptions (e.g., *"A quiet place near mountains suitable for meditation"*). The core feature is a **RAG-based Itinerary Generator** that takes selected destinations and creates a logical, day-by-day travel plan, complete with packing suggestions and budget estimates.

### 4. Target Users
- **Solo Travelers:** Looking for unique, off-the-beaten-path experiences.
- **Family Vacation Planners:** Needing structured plans that accommodate kids/seniors.
- **Travel Bloggers:** Who want to curate and share unique trip plans.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Semantic Destination Discovery**
    *   Users can search using natural language (e.g., *"Romantic dinner spots with a sunset view"*). The system uses Pinecone to search through descriptions and reviews of locations to find the best matches, rather than filtering by city/price alone.
2.  **Context-Aware Itinerary Generator (RAG)**
    *   Once a user selects a destination (e.g., "Kyoto") and interests (e.g., "Temples & Ramen"), the AI generates a detailed **3-Day Itinerary**. It retrieves specific details about opening hours and location from the database to ensure the plan is logical (e.g., grouping nearby spots together).
3.  **Smart Packing Assistant**
    *   Based on the destination's description and the user's planned activities (hiking vs. fine dining), generate a checklist of essential items to pack.
4.  **"Hidden Gem" Detector**
    *   Analyze the text of reviews for specific locations. If the AI detects phrases indicating "underrated," "secret spot," or "locals only," tag the location as a "Hidden Gem" in the UI to encourage unique exploration.
5.  **Local Cultural & Etiquette Guide**
    *   When a user finalizes a trip, use Gemini to generate a "Do's and Don'ts" card specific to that location’s culture (e.g., Tipping customs, Dress codes for temples), derived from the destination's knowledge base.

### 6. High-Level System Architecture

*   **Frontend (React):** A visually rich UI with cards for destinations, a drag-and-drop style itinerary view, and a chat interface for refining plans.
*   **Backend (Node.js + Express):** APIs to manage destination data, user saved trips, and AI generation endpoints.
*   **Database (MongoDB Atlas):**
    *   Stores: Structured Destination Data (Name, City, Price Tier, Opening Hours), User Profiles, and Saved Itineraries.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Destination Descriptions** and **User Reviews**. This enables the "Vibe Search."
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Embeddings, Itinerary generation, and Cultural advice summarization.

### 7. AI & RAG Workflow Explanation

**Phase A: Destination Ingestion**
1.  **Data Entry:** Admin adds a location: "Fushimi Inari Shrine".
2.  **Description:** "A famous shrine with thousands of vermilion torii gates, located on a wooded mountain..."
3.  **Embedding:** Node.js sends this description to Gemini to create a vector.
4.  **Storage:**
    *   *Pinecone:* Stores the vector.
    *   *MongoDB:* Stores the vector ID plus metadata (City: Kyoto, Type: Landmark, Rating: 4.8).

**Phase B: Itinerary Creation (The RAG Flow)**
1.  **User Input:** User selects "Kyoto" and types preference: *"I want to see historical sites but avoid big crowds, and I love matcha."*
2.  **Retrieval:** The system queries Pinecone for locations in Kyoto that semantically match "historical," "quiet/avoid crowds," and "matcha/tea."
3.  **Augmentation:**
    *   *Context:* (Details of the top 5 matched locations: Name, Description, Opening Hours).
    *   *Constraint:* "Create a 1-day plan. Ensure efficient travel route."
4.  **Generation:** Gemini organizes these spots into a morning/afternoon/evening schedule, adding commentary on *why* these spots fit the user's request.
5.  **Output:** React UI displays a timeline view of the itinerary.

### 8. Expected Learning Outcomes
*   Handling **Geospatial/Location Data** conceptually in an AI context.
*   Designing **Compound AI Prompts** (combining user constraints with retrieved database facts).
*   Using **Metadata Filtering** in Pinecone (searching for "Quiet spots" *only* within "Kyoto").
*   Presenting **AI-generated lists** (Itineraries) in a structured, interactive frontend component.

### 9. Constraints & Assumptions
*   **Data Set:** Students do not need to use Google Maps API. They should create a "Mock Database" of 20-30 locations (e.g., landmarks in a single city like Paris or Tokyo) to keep it contained.
*   **Images:** Use placeholder images or simple URLs for the destinations.
*   **Logic:** The AI doesn't need to know real-time traffic; it assumes standard travel times.

### 10. Optional Extensions (For Advanced Students)
