Here is a comprehensive problem statement for the **"HireSmart"** capstone project, designed for a 12-day GenAI + MERN workshop.

---

### 1. Project Title
**HireSmart: AI-Powered Intelligent Recruitment Dashboard**

### 2. Real-World Context / Business Background
In the modern recruitment landscape, HR managers and recruiters are overwhelmed by the volume of applications. Traditional Applicant Tracking Systems (ATS) rely on rigid keyword matching (e.g., searching strictly for "React" or "Python"). This often leads to rejecting high-potential candidates who may list "MERN Stack" or "Django" instead of the exact keywords. Furthermore, preparing for interviews and writing personalized rejection or offer emails is time-consuming and repetitive. Businesses need a system that understands *context*, not just keywords.

### 3. Problem Statement
Design and build **HireSmart**, a web application that streamlines the hiring process using Generative AI. The system must allow recruiters to upload resumes and Job Descriptions (JDs). Instead of simple text matching, the application should use **Semantic Search** to find the best candidates based on meaning and context. Additionally, the system must utilize a **RAG (Retrieval-Augmented Generation)** architecture to analyze how well a specific candidate fits a specific job, generate tailored interview questions based on the candidate's actual project experience, and draft professional communication automatically.

### 4. Target Users
- **Recruiters / HR Managers:** Who need to filter hundreds of resumes quickly.
- **Hiring Managers:** Who need assistance in generating technical interview questions specific to a candidate's profile.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **AI Resume Parsing & Structured Data Extraction**
    *   When a resume (text/PDF) is uploaded, use Gemini to extract key fields (Name, Skills, Experience Years, Contact Info) into a structured JSON format to auto-fill the database.
2.  **Semantic Candidate Search (Vector Search)**
    *   Allow recruiters to search using natural language (e.g., *"Find me a developer who has experience in building scalable backend systems"*).
    *   This must search through candidate embeddings in Pinecone, not just text matching in MongoDB.
3.  **RAG-Based "Fit Analysis" & Scoring**
    *   The user selects a Job Description and a Candidate. The system retrieves the full resume context and uses Gemini to generate a "Fit Score" (0-100) along with a bulleted list of "Pros" and "Cons" relative to that specific JD.
4.  **Context-Aware Interview Question Generator**
    *   Generate 5 unique technical and behavioral interview questions based *specifically* on the projects and gaps mentioned in the candidate's resume (e.g., *"I see you used MongoDB in your E-commerce project; how did you handle data consistency?"*).
5.  **Automated Email Assistant**
    *   Generate a personalized email draft (Invitation to Interview or Rejection) that references specific skills or reasons from the analysis, ensuring the candidate feels respected even if rejected.

### 6. High-Level System Architecture

*   **Frontend (React):** A dashboard to view candidate lists, upload resumes, manage Job Descriptions, and view AI insights.
*   **Backend (Node.js + Express):** Handles API requests, file processing, and orchestration between the Database and AI services.
*   **Database (MongoDB Atlas):**
    *   Stores structured data: Candidate Profile (Name, Email, parsed JSON metadata), Job Descriptions, Application Status.
*   **Vector Database (Pinecone):**
    *   Stores high-dimensional **embeddings** of the candidate's *Resume Summary* or *Experience Section*.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for generating embeddings (via embedding models).
    *   Used for text generation (Parsing, Scoring, Questions, Emails).

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion (Data Prep)**
1.  **Upload:** Recruiter uploads a resume (PDF/Text).
2.  **Extraction:** Node.js extracts raw text.
3.  **Embedding:** The backend sends the text to Google Gemini’s Embedding API to get a vector representation.
4.  **Storage:**
    *   *MongoDB:* Saves the raw text and metadata.
    *   *Pinecone:* Saves the Vector ID and Embedding.

**Phase B: Retrieval & Generation (The RAG Flow)**
1.  **User Action:** Recruiter views a Job Description and clicks "Analyze Candidate X".
2.  **Retrieval:** The system fetches Candidate X's full resume text from MongoDB.
3.  **Augmentation:** The system constructs a prompt containing:
    *   The Job Description (Context 1)
    *   The Candidate's Resume (Context 2)
    *   Instruction: "Analyze the fit between this JD and this Resume."
4.  **Generation:** This prompt is sent to the Gemini Model.
5.  **Output:** Gemini returns the Analysis, Score, and Interview Questions, which are displayed on the React Frontend.

### 8. Expected Learning Outcomes
By completing this project, students will learn:
*   How to integrate **Google Gemini APIs** into a Node.js backend.
*   The difference between **Lexical Search** (MongoDB) and **Semantic Search** (Pinecone).
*   How to implement a **RAG pipeline** to force the AI to answer based on specific data (the resume) rather than general knowledge.
*   Handling asynchronous AI operations and loading states in **React**.
*   Prompt Engineering techniques to get structured JSON outputs from LLMs.

### 9. Constraints & Assumptions
*   **File Format:** All resumes are simple text files/PDFs.
*   **Data Volume:** System should handle at least 10-20 dummy resumes for demonstration.
*   **Latency:** AI responses take time; the UI must show proper loading spinners/skeleton loaders.
*   **Cost:** All tools (MongoDB Free Tier, Pinecone Starter, Google AI Studio Free Tier) must remain within free limits.

### 10. Optional Extensions (For Advanced Students)
*   **Chat with Resume:** Add a chat window where the recruiter can ask questions directly to a specific resume (e.g., *"Does this candidate have leadership experience?"*).
*   **Resume Anonymizer:** An AI feature that strips Name, Gender, and College to promote bias-free hiring before the review stage.
*   **LinkedIn Scraper:** A browser extension or input field to scrape a public LinkedIn profile and turn it into a resume format for the system.
