### 1. Project Title
**StaySmart: AI-Powered Hotel Discovery & Booking Aggregator**

### 2. Real-World Context / Business Background
Traditional hotel booking platforms (like OYO, Booking.com, or Airbnb) rely heavily on rigid filters (Location, Date, Price, Guests). However, travelers often have specific, abstract requirements that checkboxes can't capture. A user might want *"A quiet place for remote work with strong WiFi and good coffee nearby"* or *"A budget-friendly hotel that feels luxurious and is safe for solo female travelers."* Finding these requires reading hundreds of reviews. There is a need for a platform that allows users to search by **"Vibe" and "Context"** rather than just location.

### 3. Problem Statement
Develop **StaySmart**, a hotel aggregation platform. The system should manage a database of multiple hotels, each with descriptions, amenities, and user reviews. The core feature is **Semantic Search**, allowing users to find properties using natural language descriptions of their ideal experience. Furthermore, the system must use **RAG** to answer specific pre-booking questions (e.g., *"Is the breakfast actually good?"*) by retrieving and synthesizing insights from hundreds of past guest reviews, saving the user from reading them manually.

### 4. Target Users
- **Digital Nomads:** Looking for specific work conditions (WiFi, desk, quiet).
- **Vacationers:** Searching for specific atmospheres (Romantic, Party, Zen).
- **Budget Travelers:** Who want the best value without compromising safety.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **"Vibe" Search (Semantic Discovery)**
    *   Users search naturally: *"A cozy cottage near the mountains suitable for a writer's retreat."*
    *   The system uses **Pinecone** to match this query against hotel descriptions and *aggregated review sentiments*, returning results that feel right, even if they don't explicitly say "Writer's Retreat" in the title.
2.  **Review Summarizer & Sentiment Analyst**
    *   Instead of showing 500 reviews, the AI analyzes them to generate a **"Consensus Card"**:
        *   *Pros:* "Excellent Location, Friendly Staff."
        *   *Cons:* "Noisy AC, Slow Elevator."
        *   *Vibe:* "Busy/Energetic."
3.  **The "Better Choice" Comparator**
    *   User selects two hotels (Hotel A vs. Hotel B) and asks: *"Which one is better for a family with kids?"*
    *   The AI retrieves data for both (Amenities + Reviews regarding "kids/family") and generates a comparative analysis: *"Hotel A has a pool, but reviews mention thin walls. Hotel B is rated higher for quietness and has a kids' club. Hotel B is recommended."*
4.  **Specific Policy Q&A (RAG)**
    *   Users can ask: *"Does this hotel allow unmarried couples?"* or *"Is the kitchen fully stocked?"*
    *   The system searches the specific hotel's **Policy Documents** and **Amenities List** to give a verified "Yes/No" answer with a citation.
5.  **Smart Price Value Analyzer**
    *   The AI compares the hotel's price against its amenities and average rating.
    *   It outputs a "Value Score": *"Great Deal: This hotel is $50/night but offers amenities typically found in $80/night hotels in this area."*

### 6. High-Level System Architecture

*   **Frontend (React):** A search landing page, a "Search Results" grid with AI-generated tags, and a Hotel Detail page with the "Comparator" widget.
*   **Backend (Node.js + Express):** APIs for hotel inventory, search logic, and AI processing.
*   **Database (MongoDB Atlas):**
    *   Stores: Hotel Metadata (Name, Price, Address), Structured Amenities, and Raw User Reviews.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Hotel Descriptions** and **Review Chunks**. This powers the semantic search.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Vibe matching, Summarization, Comparison logic, and Value analysis.

### 7. AI & RAG Workflow Explanation

**Phase A: Inventory Ingestion**
1.  **Data Entry:** Create 10 dummy hotels. Each has a description and 20 dummy reviews.
2.  **Review Aggregation:** Node.js concatenates the top 20 reviews for Hotel X into a single text block.
3.  **Embedding:**
    *   Embed *Description* (for factual search).
    *   Embed *Review Summary* (for vibe search).
4.  **Storage:** Pinecone stores vectors with metadata `{ hotelId: "101", city: "Manali", priceTier: "Budget" }`.

**Phase B: The Search & Compare Flow (RAG)**
1.  **User Query:** *"Remote work friendly hotels in Manali under $50."*
2.  **Retrieval:**
    *   Filter Pinecone by `city: Manali` (Metadata filtering).
    *   Vector search for "Remote work friendly, good wifi, quiet."
3.  **Ranking:** The system retrieves Top 3 matches.
4.  **Comparator Action:** User clicks "Compare Top 2".
5.  **Augmentation:**
    *   *Context A:* (Hotel A Reviews: "Wifi is fast, but construction noise nearby").
    *   *Context B:* (Hotel B Reviews: "Wifi okay, very quiet library area").
    *   *Task:* "Compare strictly on remote work suitability."
6.  **Generation:** Gemini outputs: *"Hotel A has better speed, but Hotel B is quieter. For calls/meetings, Hotel B is safer."*

### 8. Expected Learning Outcomes
*   **Hybrid Search:** Combining **Vector Search** (Vibe) with **Traditional Filters** (City/Price) in a single query (e.g., Pinecone Metadata Filters).
*   **Sentiment Aggregation:** Learning how to condense thousands of words (reviews) into meaningful AI insights.
*   **Comparative RAG:** Using AI to evaluate *two different contexts* (Hotel A vs B) simultaneously.
*   **UI/UX for Search:** Displaying AI reasoning ("Why we picked this hotel") alongside the result.

### 9. Constraints & Assumptions
*   **Data Source:** Do **not** scrape real booking sites. Create a JSON file with ~10-15 mock hotels and generate ~50 fake reviews using ChatGPT/Gemini to populate the database.
*   **Booking Flow:** No need for real payment. A simple "Book Now" button that saves a reservation to MongoDB is sufficient.
*   **Images:** Use placeholders.

### 10. Optional Extensions (For Advanced Students)
*   **Dynamic Itinerary:** Once booked, the AI suggests a 2-day itinerary *starting from that specific hotel's location*.
*   **Multilingual Reviews:** If a hotel has reviews in French/German, the AI translates and summarizes them into English for the summary card.
*   **Negotiation Bot:** A chat feature where the user can ask "Can I get a discount?", and the AI checks if the user is a "Returning Customer" (in MongoDB) to offer a 10% coupon code.
