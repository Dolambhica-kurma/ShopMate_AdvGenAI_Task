Here is problem statement #5. This project is centered on the **Marketing/Business Intelligence** domain, focusing on analyzing unstructured customer feedback.

---

### 1. Project Title
**Insightful: AI Customer Feedback Analysis Platform**

### 2. Real-World Context / Business Background
Modern companies collect vast amounts of customer feedback from various channels: app store reviews, survey responses, support chats, and social media. This unstructured text is a goldmine of information, but manually reading and categorizing thousands of comments is impossible. Product managers and marketers need an automated way to distill this "voice of the customer" into actionable insights to improve their products and marketing strategies.

### 3. Problem Statement
Design and build **Insightful**, a web dashboard that ingests raw customer feedback and uses Generative AI to perform a comprehensive analysis. The system must go beyond simple keyword counting. Using **RAG**, it will categorize feedback, identify emerging trends, and allow product managers to ask natural language questions about their customer base. The ultimate goal is to transform messy, qualitative feedback into structured, quantitative insights for data-driven decision-making.

### 4. Target Users
- **Product Managers:** Who need to prioritize features based on user requests and pain points.
- **Marketing Teams:** Who want to understand customer sentiment and find testimonials.
- **Startup Founders:** Who need a quick pulse on how their product is perceived in the market.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Feedback Ingestion & Auto-Categorization**
    *   When a user pastes a block of feedback (e.g., 20 app store reviews), the system processes each one. For each piece of feedback, it uses Gemini to extract a structured JSON object: `{ "summary": "...", "sentiment": "Positive/Negative/Neutral", "category": "Bug Report/Feature Request/Praise/Other" }`.
2.  **Semantic Search & Synthesis**
    *   A search bar allows users to ask questions like: *"What are the main complaints about the user interface?"* or *"Summarize what users love about the new checkout process."* The system uses Pinecone to find relevant feedback and Gemini to synthesize a concise, bullet-point answer.
3.  **Emerging Theme Detection**
    *   On the dashboard, display a "Top 3 Emerging Themes" card. The AI analyzes the most recent batch of feedback (e.g., last 50 entries) and identifies recurring topics of complaint or praise that weren't prominent before (e.g., "Slow loading times after recent update").
4.  **Actionable Item Generation**
    *   Allow a user to select a "Feature Request" or "Bug Report" feedback. A button "Create Action Item" sends the text to Gemini, which transforms it into a structured ticket format: `{ "title": "...", "description": "...", "user_impact": "..." }`, ready to be sent to an engineering team.
5.  **Competitive Snapshot Generator**
    *   Provide a feature where the user can paste reviews for a *competitor's* product. The AI will analyze them and generate a brief "Strengths & Weaknesses" report comparing the competitor's feedback themes to the user's own product feedback.

### 6. High-Level System Architecture

*   **Frontend (React):** A BI-style dashboard with charts for sentiment distribution, a list of categorized feedback, a powerful search interface, and modals for displaying AI-generated insights.
*   **Backend (Node.js + Express):** API for ingesting text, orchestrating the analysis pipeline with Gemini and Pinecone, and serving data to the frontend.
*   **Database (MongoDB Atlas):**
    *   Stores: The raw feedback text, alongside the structured JSON data generated by the AI (sentiment, category, summary, etc.).
*   **Vector Database (Pinecone):**
    *   Stores: High-dimensional embeddings of the *raw feedback text*. This is essential for the semantic search feature.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Text embedding, classification, summarization, and structured data generation.

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion**
1.  **Input:** User pastes 20 customer reviews into a text area.
2.  **Processing:** The Node.js backend splits the text into individual reviews.
3.  **Embedding & Storage:** For each review, it:
    *   Sends the text to Gemini to get a vector embedding.
    *   Stores the embedding in Pinecone with a reference to the review's ID.
    *   Calls Gemini again to get the structured JSON (sentiment/category) and stores both the raw text and the JSON in MongoDB.

**Phase B: Semantic Search (The RAG Flow)**
1.  **User Query:** User types into the search bar: *"Why are users canceling their subscriptions?"*
2.  **Retrieval:** The backend embeds this question and queries Pinecone to retrieve the top 5 most semantically similar customer reviews.
3.  **Augmentation:** It constructs a detailed prompt for Gemini:
    *   *Context:* (The full text of the 5 retrieved reviews).
    *   *Question:* "Based ONLY on the customer reviews provided, summarize the main reasons users are canceling their subscriptions."
4.  **Generation:** Gemini analyzes the provided context and generates a synthesized summary, which is then displayed on the React frontend.

### 8. Expected Learning Outcomes
*   Understanding how to process and analyze large blocks of unstructured text.
*   Building a multi-step AI pipeline (ingest -> categorize -> embed).
*   Using RAG to answer questions based on a custom knowledge base (the feedback data).
*   Visualizing AI-generated data in a meaningful way on a dashboard (e.g., pie charts for sentiment).
*   Prompt engineering to switch between different tasks (summarization, classification, structured extraction).

### 9. Constraints & Assumptions
*   **Data Source:** For the project, students will use publicly available data, such as copy-pasted Amazon reviews or app store reviews for a well-known app. No web scraping is required.
*   **Batch Processing:** Assume feedback is ingested in batches (e.g., 20-50 reviews at a time) to simplify the architecture.
*   **UI Focus:** The dashboard's value is in its clear presentation of AI insights. Students should focus on making the data easy to understand.

### 10. Optional Extensions (For Advanced Students)
*   **Root Cause Analysis Agent:** For a "Bug Report" theme, create an AI agent that tries to hypothesize the root cause by correlating different pieces of feedback (e.g., "Users on Android 13 are reporting crash issues related to the camera").
*   **Automated "Positive Review" Tweet Generator:** Add a button next to a "Praise" category feedback that drafts a marketing tweet thanking the user and highlighting the praised feature.
*   **Time-Series Analysis:** If timestamps are included, create a line chart showing sentiment over time to see if a recent update caused a dip or spike in user happiness.
