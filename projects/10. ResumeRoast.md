### 1. Project Title
**ResumeRoast: AI-Powered Career Coach & Resume Optimizer**

### 2. Real-World Context / Business Background
Final-year students and fresh graduates often apply to hundreds of jobs but receive zero callbacks. The main reason is not always a lack of skill, but a lack of **relevance**. Automated Applicant Tracking Systems (ATS) reject resumes that don't match the specific keywords of a Job Description (JD). Students struggle to tailor their resume for every single application, and professional career counseling is often too expensive for them.

### 3. Problem Statement
Build **ResumeRoast**, a career preparation platform. The application allows students to upload their Resume and a target Job Description. Using **RAG (Retrieval-Augmented Generation)**, the system performs a deep "Gap Analysis," highlighting exactly what the resume is missing compared to the JD. It shouldn't just list keywords; it must act as a strict "Career Coach," offering critical feedback ("roasting" weak points) and rewriting vague bullet points into impactful, metric-driven statements.

### 4. Target Users
- **University Students:** Preparing for campus placements.
- **Job Seekers:** Who are applying to multiple specific roles (e.g., specific JDs for React Developer vs. Node Developer).
- **Career Centers:** Who want to automate the initial resume review process.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **The "6-Second Scan" (AI Recruiter Persona)**
    *   Simulate a busy recruiter who scans a resume in 6 seconds. The AI provides a "First Impression" summary: *"I glanced at this and saw 'Junior Dev' but no GitHub link. Rejected."* This teaches students what stands out immediately.
2.  **ATS Score Predictor & Gap Analysis**
    *   Compare the Resume against the target JD rules used by Applicat Tracking Systems.
    *   **Output:** A Match Score (e.g., 72%) and a specific "Missing Keywords" table (e.g., *"JD asks for 'CI/CD' (High Priority) - Not found in your resume"*).
3.  **Bullet Point "Impactifier" (Metric Injection)**
    *   Users select a weak, task-based bullet point (e.g., *"Helped with database migration"*).
    *   The AI uses RAG rules to rewrite it into an *Outcome-Based* statement using the XYZ method: *"Migrated 1TB of legacy data to MongoDB (X) with zero downtime (Y), reducing query latency by 40% (Z)."*
4.  **The "Skill Bridge" Recommender**
    *   If a critical skill is missing (e.g., "Docker"), the system doesn't just flag it. It retrieves a specific, free "Nano-Project" idea or resource (e.g., *"Build a containerized Hello World app in 1 hour"*) so the student can validly add that skill to their resume by the weekend.
5.  **Context-Aware Mock Interviewer**
    *   Generate 5 technical questions based *specifically* on the claims made in the resume.
    *   *RAG Usage:* If the user lists "Built a Chat App," the AI asks: *"You mentioned using Socket.io in your Chat App; how did you handle user disconnection events?"* (Proving they actually did the work).

### 6. High-Level System Architecture

*   **Frontend (React):** A "Split View" dashboard (Resume on left, Analysis on right). A text editor for the "Roast" feedback and a "Playground" for rewriting bullet points.
*   **Backend (Node.js + Express):** Handles file uploads (PDF parsing), text processing, and AI orchestration.
*   **Database (MongoDB Atlas):**
    *   Stores: User Profiles, Saved Resumes (Text content), Past Analysis Reports, and User "Improvement History."
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Resume Segments** and **Job Description Segments**. This allows the AI to "Retrieve" the specific paragraph in the resume that matches (or fails to match) a requirement in the JD.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Comparison logic, Text Rewriting, Question Generation, and Structured JSON outputs (Scores).

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion**
1.  **Upload:** User uploads `resume.pdf`.
2.  **Parsing:** Node.js extracts raw text.
3.  **Chunking:** The text is split into sections (Education, Skills, Projects).
4.  **Embedding:** Gemini generates vectors for each section.
5.  **Storage:** Vectors stored in Pinecone with metadata `userId` and `sectionType`.

**Phase B: The Gap Analysis (RAG Flow)**
1.  **Context:** User pastes a JD for "Junior React Developer."
2.  **Query:** System chunks the JD into key requirements (e.g., "Experience with Redux").
3.  **Retrieval:** For each requirement, query Pinecone to see if the Resume has a matching section.
4.  **Augmentation:**
    *   *JD Requirement:* "Must know Redux."
    *   *Retrieved Resume Context:* "Built a To-Do app using React state." (No Redux found).
5.  **Generation:** Gemini determines this is a **Gap**.
6.  **Output:** React UI displays: *"Missing Skill: Redux. Recommendation: Add a project using Redux or mention if you learned it theoretically."*

### 8. Expected Learning Outcomes
*   **Document Comparison:** Learning how to use AI to compare two different text sources (Resume vs JD).
*   **Prompt Engineering for Tone:** Tuning the AI to be "Critical/Professional" (The Roast) vs "Helpful" (The Rewriter).
*   **PDF Parsing:** Handling real-world file inputs in Node.js.
*   **Structured Output:** Forcing the AI to return a specific "Score" and "List" rather than a generic paragraph.

### 9. Constraints & Assumptions
*   **File Format:** Focus on text-based PDFs. Students can use libraries like `pdf-parse` or `pdfjs-dist`. Avoid image-based resumes (OCR is too complex for this level).
*   **Rate Limiting:** The "Roast" feature should be limited (e.g., 3 free roasts per user) to save API tokens.
*   **Privacy:** Resumes contain PII (Phone/Email). Students should sanitize or ignore this data in the prompt.

### 10. Optional Extensions (For Advanced Students)
*   **LinkedIn Bio Generator:** Generate a punchy LinkedIn headline and "About" section based on the uploaded resume.
*   **Resume Version Control:** Allow users to save "Version A" (for Frontend roles) and "Version B" (for Backend roles) and switch between them.
*   **Success Probability:** Ask the AI to estimate the % chance of getting an interview based on the analysis (a fun, albeit subjective, metric).
