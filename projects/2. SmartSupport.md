### 1. Project Title
**SmartSupport: Intelligent Customer Service Triage & Resolution Hub**

### 2. Real-World Context / Business Background
Customer support teams in e-commerce or SaaS companies often face a deluge of support tickets ranging from "Where is my order?" to complex technical bugs. Manually reading, tagging, and assigning priority to every ticket is inefficient. Furthermore, junior agents often struggle to draft accurate responses or find the right help articles (Knowledge Base) to solve the issue, leading to inconsistent customer experiences.

### 3. Problem Statement
Develop **SmartSupport**, a help-desk dashboard for support agents. The system should automatically analyze incoming support tickets for sentiment and urgency. Instead of agents manually searching for answers, the system must use **RAG (Retrieval-Augmented Generation)** to fetch relevant "Knowledge Base (KB)" articles or similar past resolved tickets and propose a solution automatically. The goal is to reduce response time and ensure consistent quality in support replies.

### 4. Target Users
- **Support Agents:** Who need to respond to tickets quickly and accurately.
- **Support Managers:** Who need to see an overview of critical issues and customer sentiment.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Auto-Triage & Sentiment Analysis**
    *   When a new ticket arrives, use Gemini to analyze the text and output structured JSON containing: `Sentiment` (Positive/Neutral/Negative/Angry), `Priority` (High/Medium/Low), and `Category` (Billing/Technical/Feature Request).
2.  **Semantic Knowledge Base Search**
    *   When an agent opens a ticket, the system automatically queries Pinecone to find the top 3 most relevant "Knowledge Base Articles" or "FAQs" that semantically match the user's issue (e.g., matching "My screen is black" with a KB article about "Display Troubleshooting").
3.  **AI-Drafted Response (RAG)**
    *   Using the retrieved KB articles as context, use Gemini to generate a polite, professional, and technically accurate draft response for the agent to review and send.
4.  **Similar Issue Detection**
    *   Identify if this specific problem has been solved before by searching vector embeddings of *past resolved tickets*. If a match is found, show the "Previous Solution" to the agent.
5.  **Ticket Summarization & Handoff Notes**
    *   For long email threads, generate a concise "Issue Summary" so a new agent taking over the shift doesn't have to read the entire history.

### 6. High-Level System Architecture

*   **Frontend (React):** A ticketing dashboard (Kanban or List view) showing ticket status, priority tags, and a "Solver" interface with AI suggestions.
*   **Backend (Node.js + Express):** API to handle ticket creation, status updates, and AI orchestration.
*   **Database (MongoDB Atlas):**
    *   Stores: User/Agent profiles, Ticket metadata (Status, Priority, text history), and the raw text of Knowledge Base articles.
*   **Vector Database (Pinecone):**
    *   Stores embeddings of **Knowledge Base Articles** (to find answers).
    *   (Optional) Stores embeddings of **Closed Tickets** (to find historical precedents).
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for Embedding generation (text-to-vector).
    *   Used for Classification (JSON mode) and Response Generation (RAG).

### 7. AI & RAG Workflow Explanation

**Phase A: Knowledge Base Ingestion**
1.  **Admin Action:** Admin adds a generic FAQ or Help Article to MongoDB.
2.  **Embedding:** Node.js sends the Article text to Gemini to generate an embedding.
3.  **Indexing:** The embedding is stored in Pinecone with metadata pointing to the MongoDB Article ID.

**Phase B: Ticket Processing & Resolution (The RAG Flow)**
1.  **Ingest:** A user submits a complaint: "I was charged twice for the pro plan."
2.  **Analysis:** Backend sends text to Gemini -> Returns JSON: `{ priority: "High", category: "Billing" }`. MongoDB is updated.
3.  **Retrieval:** Agent opens the ticket. Backend embeds the complaint text and queries Pinecone for relevant Billing FAQs.
4.  **Augmentation:** Construct prompt:
    *   *User Complaint:* "I was charged twice..."
    *   *Retrieved Context:* (Content of "Refund Policy" and "Double Charge Fix" articles).
    *   *Task:* "Draft a polite reply explaining the refund process based on the context."
5.  **Response:** Gemini generates the draft, and the React UI populates it in the reply box for the agent to approve/edit.

### 8. Expected Learning Outcomes
*   Mastering **Prompt Engineering** for classification (getting strict JSON outputs vs free text).
*   Implementing **Semantic Search** to link user problems to documentation.
*   Understanding the **Human-in-the-loop** AI workflow (AI suggests, Human reviews).
*   Managing **CRUD operations** in MERN alongside vector operations.
*   Learning to structure data for efficient **RAG retrieval**.

### 9. Constraints & Assumptions
*   **Knowledge Base:** Students can create 10-15 dummy "Help Articles" (e.g., Shipping Policy, Reset Password, Payment Failure) to seed the database.
*   **Input Data:** Tickets can be submitted via a simple form on the frontend (simulating a user) and viewed on a separate "Agent Dashboard" route.
*   **Security:** No need for complex role-based auth; a simple toggle or login for "User" vs "Agent" is sufficient.

### 10. Optional Extensions (For Advanced Students)
*   **Multilingual Support:** Detect the language of the ticket (e.g., Spanish) and have the AI translate the incoming text to English for the agent, then translate the draft response back to Spanish.
*   **Auto-Close Agent:** If the AI determines the intent is just "Thank you" or generic feedback, it can suggest auto-closing the ticket without human intervention.
*   **Voice-to-Text:** Allow users to upload a voice note describing their issue (using a browser API), transcribe it, and then run the AI pipeline.
