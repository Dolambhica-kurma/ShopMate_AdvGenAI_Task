### 1. Project Title
**EduGap: AI-Driven Learning Content Optimizer**

### 2. Real-World Context / Business Background
Educational institutions and online course creators produce vast amounts of study material (PDFs, Lecture Notes). They also administer assessments (Quizzes/Exams). However, there is a major disconnect: when students fail a specific topic, instructors often don't know *why*. Is the concept too hard? Was the lecture material insufficient? Was the quiz question misleading? Currently, teachers have to guess where the "Knowledge Gap" lies. They need a tool that connects student performance data back to the source material to identify and fix weak links in their curriculum.

### 3. Problem Statement
Develop **EduGap**, a content intelligence dashboard for instructors. The system ingests **Course Materials** (PDFs/Text) and **Student Assessment Data** (Quiz Scores/Logs). Using **RAG**, it analyzes topics where students consistently underperform. The core innovation is the **"Content Gap Detector"**: the AI retrieves the specific section of the course material corresponding to the failed questions and critiques *itself*—determining if the material was too vague, missing examples, or overly complex—and then suggests specific content improvements.

### 4. Target Users
- **Course Instructors:** Who want to improve their lecture notes based on data.
- **L&D Managers:** In corporate training, ensuring employees actually learned the policy.
- **Content Creators:** Optimizing their tutorials for better engagement.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **Performance-Triggered Gap Detection**
    *   *Function:* Analyze uploaded quiz logs. If >50% of students fail questions tagged "Module 3", the system automatically flags it as a **"Weak Topic"** and triggers the RAG analysis.
2.  **Content Self-Critique Engine (RAG)**
    *   *Function:* The AI retrieves the specific section of the course material corresponding to the failed topic. It then critiques *itself*: "Does this text explain the concept clearly? Is it missing examples?"
3.  **Gap-Based Content Generator (The "Fixer")**
    *   *Function:* Once a flaw is identified (e.g., "Explanation is too abstract"), the AI generates a **"Remedial Patch"**—a simplified explanation, a real-world analogy, or a code walkthrough to replace the confusing original text.
4.  **Student-Specific "Bridge" Notes**
    *   *Function:* For a student who failed, generate a personalized study guide. It uses **"Explain Like I'm 5" (ELI5)** logic to rewrite complex jargon into simple terms, targeting only their specific failure points.
5.  **Targeted Remedial Quiz Generator**
    *   *Function:* To verify improvement, the system auto-generates a new set of 5 MCQs based *specifically* on the newly generated remedial content, ensuring the new material is properly tested.


### 6. High-Level System Architecture

*   **Frontend (React):** A dashboard showing "Course Health" (Charts), a "Gap Report" list, and a Content Editor to review/apply AI suggestions.
*   **Backend (Node.js + Express):** APIs to handle file uploads, quiz score ingestion, and AI processing.
*   **Database (MongoDB Atlas):**
    *   Stores: Student Profiles, Quiz Results (Question-wise breakdown), and Course Metadata.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of the **Course Material (Text Chunks)**. This allows the system to find *where* a topic was taught.
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Self-Correction/Critique, Content Generation, and Simplification.

### 7. AI & RAG Workflow Explanation

**Phase A: Ingestion (Course & Data)**
1.  **Upload Material:** Instructor uploads `Java_Course_Notes.pdf`.
2.  **Indexing:** System chunks text by topic and stores vectors in Pinecone.
3.  **Upload Scores:** Instructor uploads a CSV of quiz results (or students take a quiz on the platform).
4.  **Analysis:** Node.js calculates that "Topic: Polymorphism" has an average score of 35% (FAIL).

**Phase B: The Gap Analysis (RAG Flow)**
1.  **Trigger:** System detects "Polymorphism" is a Weak Topic.
2.  **Retrieval:** Query Pinecone for "Polymorphism definition examples".
3.  **Augmentation:**
    *   *Context:* (Retrieved text from notes: "Polymorphism is the ability of an object to take on many forms.").
    *   *Evidence:* "Students are failing to identify runtime vs compile-time polymorphism."
    *   *Task:* "Critique the Context. Does it explain runtime vs compile-time differentiation clearly?"
4.  **Generation:** Gemini output: *"Critique: The notes define the concept broadly but do NOT explain the difference between Overloading and Overriding, which caused the confusion."*
5.  **Resolution:** Gemini generates a new paragraph explaining "Overloading vs Overriding" with a code example.

### 8. Expected Learning Outcomes
*   **Reverse RAG:** Instead of answering a question *from* text, using the text to explain *why* a question was answered poorly.
*   **Data-Driven AI:** Triggering AI prompts based on statistical thresholds (e.g., if Score < 50%).
*   **Content Evaluation:** Using LLMs to act as an "Editor" or "Critic."
*   **Visualizing Gaps:** Connecting quantitative data (scores) with qualitative data (text content) in the UI.

### 9. Constraints & Assumptions
*   **Input Data:** Students can generate dummy "Quiz Results" (JSON/CSV) where specific question IDs are marked as "Incorrect" to simulate a class failure.
*   **Content Scope:** Use text-based PDFs. Don't try to analyze video lectures (too complex).
*   **Granularity:** The analysis works best if the course notes have clear headings (e.g., "Chapter 1", "Chapter 2").

### 10. Optional Extensions (For Advanced Students)
*   **Difficulty Scalar:** The AI analyzes the quiz questions themselves. It might say: *"The material is fine, but Question 4 was phrased ambiguously. Suggest rewriting the question instead of the notes."*
*   **Trend Analysis:** Track if the "Content Patch" actually improved scores in the next quiz.
*   **Heatmap UI:** Visualizing the PDF document with "Red Zones" (sections that confuse students) and "Green Zones" (sections students understood well).
