### 1. Project Title
**EventMinds: AI-Powered Event Planner & Vendor Concierge**

### 2. Real-World Context / Business Background
Organizing an event—whether it’s a college fest, a tech hackathon, or a wedding—is chaotic. Organizers struggle to find the right vendors (DJs, Caterers, Decorators) that fit their specific budget and "vibe." Contracts are signed without reading the fine print (leading to hidden costs), and creating a minute-by-minute schedule ("Run of Show") is a manual, error-prone process. There is a need for a centralized dashboard that not only aggregates vendors but actively helps plan the logistics.

### 3. Problem Statement
Develop **EventMinds**, an event management workspace. The system allows organizers to create an event and manage vendors. The core feature is **Semantic Vendor Discovery**, allowing users to find service providers based on abstract descriptions (e.g., *"A photographer who captures candid, moody shots"*). Additionally, the system must use **RAG** to analyze **Vendor Contracts** to flag hidden clauses and use Generative AI to automatically draft the event itinerary and invitation content.

### 4. Target Users
- **Student Committees:** Organizing college fests/hackathons.
- **Wedding Planners:** Managing multiple clients.
- **Corporate HR:** Planning office parties.

### 5. Core AI Features
The project must include the following 5 AI-driven features:

1.  **"Vibe-Match" Vendor Search (Semantic Search)**
    *   *Input:* "I need a caterer who specializes in spicy street food and has a rustic stall setup."
    *   *Tech:* The system uses **Pinecone** to search through vendor portfolios and reviews. It retrieves matches based on the *style* and *specialty*, not just keyword tags like "Caterer."
2.  **Contract "Gotcha" Detector (RAG)**
    *   *Input:* User uploads a PDF contract from a DJ or Venue.
    *   *Tech:* The AI analyzes the document against a "Standard Fair Practice" knowledge base.
    *   *Output:* It highlights risky clauses: *"Warning: Clause 4.2 allows the DJ to cancel within 24 hours without a refund."*
3.  **"Run of Show" Generator (Itinerary)**
    *   *Input:* Event Type (e.g., "Hackathon"), Duration (24 Hours), and Key Activities (Opening Ceremony, Lunch, Coding, Judging).
    *   *Tech:* Gemini generates a detailed, time-stamped table (Schedule) ensuring logical flow (e.g., "Lunch should be at 1 PM," "Buffer time added for Judging").
4.  **Theme & Moodboard Creator**
    *   *Input:* "Cyberpunk theme with neon lights."
    *   *Tech:* The AI generates a detailed **Visual Description Prompt** that the user can use (or the app can pass to an image generation API if available) to visualize the decoration. It also suggests a color palette (Hex codes).
5.  **Smart Budget Allocator**
    *   *Input:* Total Budget (e.g., ₹5,00,000) and Priorities (e.g., "Food is high priority, Decor is low").
    *   *Tech:* The AI suggests a split: *"Allocate ₹2L for Catering, ₹50k for Sound, etc."* based on industry standard ratios stored in the database.

### 6. High-Level System Architecture

*   **Frontend (React):** A dashboard with "Vendor Marketplace," "Contract Reviewer" (PDF View), "Schedule Builder" (Timeline UI), and "Budget Pie Chart."
*   **Backend (Node.js + Express):** APIs for vendor management, file processing, and AI orchestration.
*   **Database (MongoDB Atlas):**
    *   Stores: Vendor Profiles, Event Details, Schedules, and User Budgets.
*   **Vector Database (Pinecone):**
    *   Stores: Embeddings of **Vendor Bio/Reviews** (for search) and **Legal Contract Clauses** (for RAG analysis).
*   **AI Engine (Google AI Studio - Gemini):**
    *   Used for: Search intent, Legal analysis, Schedule generation, and Budget logic.

### 7. AI & RAG Workflow Explanation

**Phase A: Vendor Onboarding**
1.  **Data Entry:** Create 10 dummy vendors (e.g., "DJ Snake", "Royal Catering").
2.  **Description:** "Specializes in EDM and Bollywood... High energy... Expensive."
3.  **Embedding:** Gemini creates vectors for these descriptions.
4.  **Storage:** Pinecone stores vectors with metadata `{ type: "DJ", price: "$$$" }`.

**Phase B: Contract Analysis (The RAG Flow)**
1.  **User Action:** Uploads `Venue_Agreement.pdf`.
2.  **Processing:** Node.js extracts text.
3.  **Retrieval:** The system queries Pinecone for "standard cancellation policies" or "hidden fees" to see what *normal* contracts look like (optional comparison) or simply uses the text as context.
4.  **Augmentation:**
    *   *Context:* (The uploaded contract text).
    *   *Prompt:* "Analyze this contract for risk. Flag any clause regarding 'Cancellation' or 'Overtime fees'."
5.  **Generation:** Gemini returns: *"Clause 9 says they charge double for every hour past 10 PM. This is non-standard."*

### 8. Expected Learning Outcomes
*   **Marketplace Logic:** Building a search that matches Buyers (Organizers) with Sellers (Vendors).
*   **Legal RAG:** Using AI to protect users from bad contracts.
*   **Structured Generation:** Getting the AI to output a Schedule (Time/Activity) that fits into a UI timeline component.
*   **Financial Logic:** Using AI to perform math/estimations (Budgeting) based on qualitative inputs.

### 9. Constraints & Assumptions
*   **Vendors:** No need for a real vendor portal. Students can seed the database with 20 mock vendors.
*   **Contracts:** Use 3-4 sample text files as "Contracts" to test the analyzer.
*   **Images:** The "Moodboard" feature can just generate *text descriptions* of images if integrating an Image Gen API (like OpenAI DALL-E) is out of scope/budget.

### 10. Optional Extensions (For Advanced Students)
*   **Email Invitation Generator:** Drafts custom emails for Sponsors, Judges, and Attendees based on the Event Description.
*   **RSVP Analyzer:** Upload a CSV of RSVP responses (e.g., "Vegetarian", "Vegan"). The AI summarizes the catering needs (e.g., "You need 40 Veg plates and 5 Vegan plates").
*   **Chat with Vendor:** A chat interface where the user talks to an "AI Agent" representing the vendor (answering questions based on the vendor's profile).
